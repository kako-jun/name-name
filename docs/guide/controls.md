# 操作ガイド

Name×Name の操作方法（ノベルプレイヤー + RPG プレイヤー）。
エディタ画面の使い方は [エディタガイド](./editor.md) を参照。
特定シーン・状態を URL／コードで再現する開発者向けの手順は [デバッグガイド](./debugger.md) を参照。

## ノベルプレイヤー

### キーボードショートカット

| キー | 動作 |
|---|---|
| Space | 次のテキストへ |
| Enter | 次のテキストへ |
| → (右矢印) | 次のテキストへ |
| ← (左矢印) | 前のテキストへ |
| S | セーブメニューを開く |
| L | ロードメニューを開く |
| B | バックログの表示/非表示を切り替え |
| Escape | 開いているオーバーレイを閉じる |

### マウス・タッチ操作

| 操作 | 動作 |
|---|---|
| クリック / タップ | 次のテキストへ |
| シークバーをクリック | クリックした位置にジャンプ |

### テキスト送り

ダイアログ（台詞）とナレーションはカノソ方式で表示される:
1. テキストが一瞬で全文表示される
2. クリック / Space / Enter / → で次のテキストに進む
3. ← で前のテキストに戻る

背景変更・BGM再生・暗転などの演出イベントはユーザー操作なしで自動実行される。

作品の frontmatter `dialog_style: "novel"`（全画面ノベル）では送りが**文単位（息継ぎ単位）**になる (#292)。クリック / タップは句点・感嘆符・疑問符で割った文を 1 つずつ送り、既出の文は同じページに溜まる。文末の右に出るクリッカーは種別で形が違う: `▼` は同じページに次の文がある（続き送り）、`❯` はそのページの最後の文＝次のクリックで改頁。ページが尽きたら次のテキストへ進む。

さらに、脚本の本文中に書かれた**単独行 `---`** は手動の強制改頁になる (#292 Phase 2)。自動改頁（収まる行数で区切る）より早くページを切りたいときに作者が入れる印で、`---` を境に続きの文が新しいページの先頭から始まる。`---` を含まない脚本は文単位送りの挙動そのまま（変化なし）。

`dialog_style: "adv"`（既定・下部 ADV 箱）は従来どおり 1 クリックで次のテキストに進む（`---` の手動改頁・文単位送り・クリッカー 2 種は novel 専用）。

### セーブ/ロード

#### セーブ

1. **S** キーを押す（またはセーブボタンをクリック）
2. セーブメニューが表示される（3スロット）
3. 保存したいスロットをクリック
4. 現在の状態がスロットに保存される

#### ロード

1. **L** キーを押す（またはロードボタンをクリック）
2. ロードメニューが表示される
3. 読み込みたいスロットをクリック
4. 保存時の状態が復元される（背景・BGM・立ち絵・テキスト位置）

#### データのエクスポート/インポート

セーブデータを JSON ファイルとしてエクスポート・インポートできる。別のブラウザや端末にデータを移行する場合に使用。

### バックログ

1. **B** キーを押す
2. これまでに表示されたダイアログ・ナレーションの一覧が表示される
3. **↑** / **↓** キーまたはマウスホイールでスクロール
4. もう一度 **B** を押すか **Escape** で閉じる

### シークバー

画面下部のプログレスバーで、ゲーム全体の進行位置を確認・操作できる。

- バーの塗りつぶし部分が現在位置を示す
- バーの任意の位置をクリックすると、その位置にジャンプする
- 背景・BGM・立ち絵などの状態も自動的に復元される

### 選択肢

選択肢が表示されたら:
1. 選択肢のボタンが画面に表示される
2. クリック / タップで選択
3. 選択に応じてフラグが設定され、対応するシーンにジャンプする

選択肢が多くて画面に収まりきらない場合だけ、選択肢一覧は画面内でスクロールできる表示になる。マウスホイールまたはタッチドラッグで上下に動かし、すべての選択肢を確認して選べる。少数の選択肢は従来どおり中央寄せで表示される。

### オーバーレイの閉じ方

セーブメニュー・ロードメニュー・バックログなどのオーバーレイは **Escape** キーで閉じられる。

## RPG プレイヤー

PixiJS ベースのタイルマップ RPG モード。`gameData` を渡さない場合はサンプルデータが読み込まれてそのまま動作する。

レンダラーはシーン側の `[view=...]` 指定に応じて切り替わる:
- `[view=topdown]`（デフォルト）: 見下ろし型（TopDownRenderer）
- `[view=raycast]`: 一人称レイキャスト型（RaycastRenderer）

### 見下ろし型（TopDown）のキーボード操作

| キー | 動作 |
|---|---|
| ↑ / W | 上に 1 マス移動 |
| ↓ / S | 下に 1 マス移動 |
| ← / A | 左に 1 マス移動 |
| → / D | 右に 1 マス移動 |
| Enter / Space | 正面の NPC に話しかける / 会話ウィンドウを閉じる |

### レイキャスト型（Raycast）のキーボード操作

一人称視点。連続移動＋回転でファーストパーソン・ダンジョン探索感覚。

| キー | 動作 |
|---|---|
| ↑ / W | 前進 |
| ↓ / S | 後退 |
| ← / A | 左回転 |
| → / D | 右回転 |
| Q | 左ストレイフ（真横に移動） |
| E | 右ストレイフ（真横に移動） |
| PageUp | 上を見る（pitch +、最大 ±0.4 rad ≒ ±23°） |
| PageDown | 下を見る（pitch −） |
| Z | ジャンプ（最高約 0.375 タイル分、滞空 0.5 秒。空中での再ジャンプ不可） |
| Enter / Space | 正面タイルの NPC に話しかける / 会話ウィンドウを閉じる |

### 移動と通行判定

- **見下ろし型**: 移動はグリッド単位。1 マス移動につき約 150ms のアニメーションが入り、その間は次の入力を受け付けない
- **レイキャスト型**: 連続移動（約 3 タイル/秒、回転 約 3 rad/秒）。キーを押し続けると移動し続ける
- 木（TREE）と水（WATER）のタイル、および NPC のいるタイルには入れない（両モード共通）
- レイキャスト型では XY 分離で壁ずり衝突する（壁に斜めに当たると沿って滑る）
- マップ端を越えることもできない

### 会話

プレイヤーの向いている先のタイルに NPC がいるとき、Enter / Space で会話が開始し、画面下部にウィンドウが表示される（NPC 名と本文）。再度 Enter / Space で閉じる。会話ウィンドウ表示中は移動入力を受け付けない。両モードで同じダイアログ UI を使う。

メニューの「はなす」から会話を開始した場合、選択した tap が直後の canvas tap として canvas に届いてダイアログが即時に閉じる/進む事故を防ぐため、二段構えの防御を入れている:

1. `handleMenuSelect` の入口で `suppressNextTouchTap(DIALOG_JUST_SHOWN_GUARD_MS)` を再延長する（同一 PointerEvent シーケンス内の漏れ tap を確実に弾く一次防御）
2. `handleTap` 側で `dialogBox.isJustShown(DIALOG_JUST_SHOWN_GUARD_MS)` が真のときは close / advance / typewriter skip を全て無視する（時刻ベースの保険、suppressNextTouchTap が漏れたケース）

ガード幅は `DIALOG_JUST_SHOWN_GUARD_MS = 400 ms`（`tapMaxDuration = 350 ms` と `suppressNextTouchTap` のデフォルト 250 ms を覆える値）。EventRunner の Dialog / Narration コマンドで `dialog.show` された直後も同じガードが効く（DialogBox 側で時刻を持っているため）。

### スマホ・タッチ操作（両モード共通）(#178)

オンスクリーンの仮想パッドは出さず、ジェスチャだけで操作する:

| 操作 | 動作 |
|---|---|
| スワイプ ↑ | TopDown: 上に 1 マス移動。Raycast: 正面に 1 タイル前進 |
| スワイプ ↓ | TopDown: 下に 1 マス移動。Raycast: 1 タイル後退 |
| スワイプ ← / → | TopDown: 左右に 1 マス移動。Raycast: 90° 回転（瞬間スナップではなく約 0.16 秒のアニメーション補間。連打すると 180° / 270° と残角度が積み上がって連続回転する。残量上限は ±2π。旋回アニメ中に前進・後退すると残角度を即時 flush してから移動する。ダイアログ / メニュー表示中も補間自体は進む） |
| シングルタップ | メニューを開閉する。ダイアログ表示中はテキスト送り（typewriter スキップ → 閉じる） |
| メニュー項目を直接タップ | その項目を即実行（カーソル移動 → 決定の二段操作はしない） |
| メニュー外のタップ | メニュー表示中なら閉じる |
| メニュー表示中のスワイプ | メニューを閉じる（誤タッチ救済） |
| TopDown の連続スワイプ | 移動アニメ中の入力は最後の 1 件をキューして取りこぼさない |

メニューは **DQ4 ファミコン版相当の左上 8 コマンドウィンドウ** (#171)。普段は隠れていてタップで出る、文字を直接タップして即決定（カーソル移動なし）。
配置は FC 版 DQ4 と同じ 2 列 x 4 行:

`はなす / じゅもん`
`つよさ / どうぐ`
`そうび / さくせん`
`とびら / しらべる`

| コマンド | サブメニュー | 振る舞い |
|---|---|---|
| はなす | なし | 正面 NPC に話しかける（Enter/Space と同じ） |
| どうぐ | アイテム一覧 | (#174 で実装) |
| しらべる | なし | 足元 / 正面を調べる（#173 戦闘内 examine とは別途実装） |
| つよさ | パーティメンバー | (#175 で実装) |
| さくせん | バッチリがんばれ / いのちだいじに / じゅもんつかうな | (#173 戦闘 AI と連動) |
| じゅもん | パーティメンバー → 呪文一覧 | (#174 で実装) |
| そうび | パーティメンバー | (#175 で実装) |
| とびら | なし | 鍵付きドアを開く（#173 鍵アイテムと連動） |

サブメニューはメインの右隣に独立パネルで開く（重なり方は実装が DQ4 ファミコン版に準拠する形で後追い調整予定）。

仮想パッド・タップで歩く方式は採用しない（Issue #178 の方針）。

### ミニマップ (#149)

**Raycast モード（一人称視点）** の画面右上に **2D 全体俯瞰ミニマップ**を半透明で重ねる。一人称で迷子にならないための助け。

- マップ全体（壁 = 暗緑/青、床 = 緑/茶）を縮小表示
- プレイヤーは **赤い三角形**（向いている方向の頂点が前向き）
- NPC は NpcData の色で **小さな丸ドット**

サイズは既定 120×120 px（`max(mapW, mapH)` に合わせて 1 タイル px が決まる）。TopDown モードは元々画面全体が俯瞰図なのでミニマップは出さない（重複するため）。

**自動非表示**: ダイアログ表示中（NPC 会話）/ メニュー表示中（DQ4 8 コマンド）はミニマップを自動的に隠す。イベントシーン・戦闘中の視覚的ノイズを避けるため。探索中だけ表示される。

将来拡張: `[minimap=on|off]` のシーン属性、Per-game 既定の切替、戦闘モード（#173）への自動連動、表示/非表示切替時のフェードアニメーション。

### 戦闘プロト (#173, Phase 1)

Raycast モードで DQ4 メニューの **「しらべる」** をタップすると、テスト用の戦闘プロトが起動する（暫定経路、本来は #172 確率エンカウントで自動発火）。

レイアウト: フィールドを半透明黒で暗くした上に、敵スプライト枠（中央上部、現状はテキストのみ）+ 4 コマンドウィンドウ（たたかう / じゅもん / どうぐ / にげる）+ 戦闘ログ（最新 4 行）を重ねる。Undertale 風 + DQ1/DQ5 風（背景フィールドが透けて見える）。

| コマンド | 動作 |
|---|---|
| たたかう | DQ 式の通常攻撃。ダメージ = max(1, atk - def/2) + rand(0..atk/4) |
| じゅもん / どうぐ | Phase 1 では未実装ログのみ（パーティ #175 で本実装） |
| にげる | 50% で escaped、失敗で敵ターン |

戦闘終了（victory / defeat / escaped）で自動的に画面を畳む。経験値 / ゴールドは集計してログに出すが、レベルアップ判定は #175 まで保留。
