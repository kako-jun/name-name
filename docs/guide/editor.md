# エディタガイド

Name×Name のエディタ画面（ノベル／RPG 2タブ構成）の使い方。

## 画面構成

エディタ画面はヘッダー直下にタブがある:

| タブ | 用途 |
|---|---|
| ノベル | `CanvasEditor` / `NovelPlayer` を使ったノベルシーンの編集・プレイ確認 |
| RPG | `MapEditor` / `NPCEditor` / `RPGPlayer` を使った RPG シーンの編集・プレイ確認 |

## RPG タブ

RPG タブを開くと、2段のツールバーが表示される。

### 1段目: サブタブ（マップ / NPC / プレイ）

RPG の編集対象を切り替える。

| ボタン | 内容 |
|---|---|
| マップ | `MapEditor` でタイル（床・草・木・水）を塗る |
| NPC | `NPCEditor` で NPC の配置・名前・色・セリフ・スプライト・歩行フレーム数・向きを編集する |
| プレイ | `RPGPlayer` で現在のシーンをプレイして確認する |

### 2段目: シーン選択 + 視点切替ツールバー

RPG シーン（= `RpgMap` ブロックを持つシーン）が存在するときのみ表示される。

#### シーン選択

ドキュメント内の RPG シーンが2つ以上ある場合、`<select>` で編集対象を切り替えられる。
1つ以下の場合は現在のシーン名を読み取り専用のラベルとして表示する。

シーン選択を変更すると、マップ・NPC・プレイの各ビューが選んだシーンの内容で再描画される。

なお、Markdown を直接編集してシーン ID を変更すると、エディタ側は選択状態を失い先頭の RPG シーンに戻る。

#### 視点切替

「視点」セレクトで RPG プレイ時のレンダラーを切り替える。

| 選択 | レンダラー | 対応する Markdown |
|---|---|---|
| 見下ろし | `TopDownRenderer`（2D見下ろし） | `[view=topdown]` または省略 |
| レイキャスト | `RaycastRenderer`（一人称） | `[view=raycast]` |

視点を変更すると即座に Markdown に書き戻され、次回ロード時にも保持される
（`applyRpgProjectToDoc` が該当シーンの `view` を書き換える）。

なお、`MapEditor` は編集優先のため常に見下ろし型で描画される。視点設定は
「プレイ」サブタブでの挙動にのみ影響する。

### NPC のスプライト設定

NPC サブタブには、NPC を選択すると「見た目の設定」パネルが表示される。スプライト・フレーム数・向きをその場で編集でき、変更は自動保存（Markdown へ書き戻し）される。

| 項目 | 内容 |
|---|---|
| スプライト | スプライトシートへの相対パス。空欄なら色付き四角で描画。`__demo` を指定すると内蔵の手続き生成スプライト（頭・胴体・足）が使われ、リポに画像を置かずに動作確認できる。**パスに空白を含むと入力欄が赤枠になり、追加ボタンが無効化される**（Markdown 属性が空白区切りで parse されるため） |
| フレーム数 | 歩行アニメのフレーム数（方向あたり）。1〜4、空欄時はレンダラーのデフォルト（2, ドラクエ式の足踏み） |
| 向き | NPC のアイドル時の向き。`下`（既定）/ `左` / `右` / `上` |

「+ NPC追加」フォームにも同じ 3 項目がある。追加時に空のままでも後から編集パネルで変更できる。

### マップエディタのタブ（タイル / 壁高さ / 床高さ / 天井高さ）

`MapEditor` にはタブバーが4つある:

| タブ | 編集対象 | fallback（未指定セル） |
|---|---|---|
| タイル | `tiles`（草地・道・木・水） | - |
| 壁高さ | `wallHeights`（壁の高さ倍率） | `1.0` |
| 床高さ | `floorHeights`（踏み込み時のカメラ上昇） | `0.0` |
| 天井高さ | `ceilingHeights`（ジャンプ時の頭ぶつけ判定） | `1.0` |

高さタブを開くと、対応する 2 次元配列が `undefined` の場合は即座に fallback 値で全セルが初期化されて Markdown に書き戻される（タブを開いた瞬間の保存）。

#### 高さパレット

高さタブのツールバーはプリセット値のボタン列になる:

- 壁・天井: `0` / `0.5` / `1` / `1.5` / `2`
- 床: `0` / `0.25` / `0.5` / `0.75` / `1`

右側にカスタム数値入力（`step=0.25`）と「選択」ボタンがあり、プリセット以外の値も塗れる。

#### セルの表示

各セルは HSL 色で高さを可視化する:

- 壁: 青系（`hsl(200, 60%, L)`、値が大きいほど暗い）
- 床: 茶系（`hsl(30, 50%, L)`）
- 天井: 紫系（`hsl(280, 40%, L)`）
- `0` は薄グレー（段差なし / 非表示の視覚的区別）

セル中央に数値オーバーレイ（白文字＋黒 text-shadow）。編集中でない他レイヤは半透明（opacity 0.3）で重ね表示され、ツールバーの「表示」チェックボックスで壁・床・天井それぞれを on/off できる。

#### ペイント操作

クリック＋ドラッグで選択中の値を塗る。`tiles` タブと同じ useState + onMouseDown/onMouseEnter パターン。

#### プレビュー

`Raycastでプレビュー` ボタンで即座に反映を確認できる。高さタブ編集中も `MapEditor` 下部のプレビューボタンは常に利用可能。

### マップ編集時のプレビューボタン

`MapEditor` のタイルパレット右側に 2 つのプレビューボタンがある:

| ボタン | 動作 |
|---|---|
| Raycastでプレビュー | 現在編集中の RPG プロジェクトを `RaycastRenderer`（一人称）で試写する |
| 見下ろしでプレビュー | 同じく `TopDownRenderer`（2D見下ろし）で試写する |

どちらもモーダル（画面 90% サイズ）として開く。閉じる方法は 3 通り:
右上の「閉じる」ボタン、Esc キー、モーダル外側（暗転部分）クリック。
シーンの `[view=...]` 設定は変更しないため、プレビュー専用の一時的な確認手段として使える。

## RPG シーンが無いプロジェクト

まだ RPG シーンを持たないプロジェクトを開いた場合、RPG タブには
「+ RPGシーンを追加」ボタンだけが表示される。クリックすると
`20x15` の空マップ（外周 = 木）と player のスケルトンが生成され、
新しいシーンとして Markdown に挿入される。
